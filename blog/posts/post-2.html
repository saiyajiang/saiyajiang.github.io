Dota 2 游廊地图入门
从零开始制作自定义地图

2026年4月15日 约 12 分钟阅读
#Dota2#游廊#Lua#地图制作

Dota 2 的游廊（Arcade）系统允许任何人制作并发布自定义游戏地图。从经典的"嘉年华"到各种创意玩法，游廊地图已经是 Dota 生态中不可或缺的一部分。本文介绍如何从零开始入门地图制作。

环境准备

安装 Dota 2 Workshop Tools

在 Steam 的 Dota 2 属性页面中，找到 DLC 标签，勾选安装 Dota 2 Workshop Tools DLC。安装完成后，在游戏启动选项中选择"进入地图编辑器"即可打开工具套件。

工具结构

Workshop Tools 包含以下核心组件：

  • Hammer：地图地形编辑器（类似 CS 的 Hammer Editor）
  • Asset Browser：资源浏览器，可以查找所有可用的模型、材质
  • Panorama UI：自定义 HUD 界面的前端框架（HTML/CSS/JS）
  • KV 编辑器：用于修改物品、技能等游戏数据

地图的基本结构

一个 Dota 2 自定义地图项目位于：

Steam\steamapps\common\dota 2 beta\game\dota_addons\你的地图名称\

目录结构如下：

your_addon/
├── maps/           # .vmap 地图文件
├── scripts/
│   ├── vscripts/   # Lua 游戏逻辑脚本
│   └── npc/        # KV 文件（英雄/物品数据覆盖）
├── resource/       # 本地化文本
└── panorama/       # 自定义 UI

Lua 脚本入门

游廊地图的游戏逻辑完全由 Lua 脚本驱动。入口文件是 vscripts/addon_game_mode.lua。

最简单的初始化

-- addon_game_mode.lua
function Precache(context)
  -- 预加载资源
end

function Activate()
  GameRules:GetGameModeEntity():SetThink("OnThink", GameRules:GetGameModeEntity(), 1)
  ListenToGameEvent("game_rules_state_change", OnStateChange, nil)
end

function OnStateChange()
  local state = GameRules:State_Get()
  if state == DOTA_GAMERULES_STATE_GAME_IN_PROGRESS then
    -- 游戏开始时的逻辑
    print("游戏开始！")
  end
end

监听事件

Dota 2 提供了丰富的游戏事件系统。常用的事件包括：

  • npc_spawned — 单位生成
  • entity_killed — 单位死亡
  • dota_player_gained_level — 玩家升级
  • dota_item_purchased — 购买物品
ListenToGameEvent("entity_killed", function(keys)
  local unit = EntIndexToHScript(keys.entindex_killed)
  if unit ~= nil and unit:IsHero() then
    print("英雄死亡：" .. unit:GetName())
  end
end, nil)

下一步

学习地图制作最快的方式，是找一个已有的开源地图，把代码读懂，然后改它。
